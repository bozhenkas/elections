## 0. Правила по комментариям

НЕ удалять:
- комментарии с пометкой TODO, FIXME, HACK, NOTE
- комментарии начинающиеся с "// dev:", "// backend:", "// api:"
- JSDoc блоки в файлах src/api/
- любые комментарии явно адресованные другим разработчикам

УДАЛЯТЬ:
- закомментированный код
- очевидные комментарии типа "// импорт компонента", "// стейт для модалки"
- console.log и debugger
- console.error КРОМЕ файлов в src/api/ и компонентов ErrorBoundary
- автосгенерированные комментарии от ИИ ("// this function handles...")

ДОБАВИТЬ:
- короткую документацию для сложных мест на русском, с маленькой буквы, сухо
- пример: // пересчёт процентов после каждого голоса, округление до десятых
- пример: // debounce 300ms чтобы не слать повторные запросы при быстрых кликах
- НЕ писать документацию к простым вещам

## 1. Структура проекта

Проверь и при необходимости реорганизуй:
- Компоненты в src/components/, каждый в своей папке (Component/index.jsx + Component.module.css)
- Хуки в src/hooks/
- Стор (zustand/context) в src/stores/ — это НЕ хуки, не переносить в hooks
- API-заглушки в src/api/
- Моковые данные в src/mock/
- Константы (magic numbers, таймауты, лимиты) в src/constants.js
- Стили-токены в src/styles/tokens.css
- Локали в src/locales/
- Утилиты в src/utils/
Удали неиспользуемые файлы, дублирующиеся стили, пустые папки.

## 2. Clean code

Для КАЖДОГО файла:
- Удали неиспользуемые импорты, переменные, функции, пропсы
- Переименуй нечитаемые переменные (e, d, tmp, data2) в понятные
- Разбей функции длиннее 30 строк на подфункции
- Вынеси magic numbers в src/constants.js
- Замени вложенные тернарники на early return или отдельные компоненты
- Убедись что каждый компонент делает одну вещь

## 3. React best practices

- Каждый компонент — функциональный
- Пропсы деструктурированы в сигнатуре функции
- Все списки имеют стабильный key (не index)
- useEffect: КАЖДЫЙ должен иметь массив зависимостей.
  Пустой [] допустим для однократного запуска при маунте.
  Отсутствующий массив — недопустим.
- useEffect: cleanup где нужно (таймеры, подписки, AbortController)
- Нет лишних ре-рендеров: memo/useMemo/useCallback только где реально нужно
- Стейт поднят на минимально необходимый уровень
- Нет prop drilling глубже 2 уровней — использовать context
- Добавь ErrorBoundary компонент для graceful обработки ошибок

## 4. CSS / стили

- Повторяющиеся значения вынеси в CSS-переменные в tokens.css
- Убери !important
- Убери неиспользуемые классы
- Проверь что нет inline-стилей которые должны быть в CSS
- Проверь адаптив: все breakpoints консистентны (использовать миксины или 
  константы в JS, не CSS-переменные в @media — они там не работают)
- Убедись что шрифты Serpantin и Decor используются только по правилам 
  из CLAUDE.md

## 5. Безопасность

- Проверь все dangerouslySetInnerHTML — должна быть санитизация
- Токен авторизации: в проде хранить в httpOnly cookie. 
  В текущих моках допустимо хранить в памяти (useState/context), 
  НЕ в localStorage. Пометь комментарием: // dev: в проде заменить на httpOnly cookie
- API-заглушки не должны содержать захардкоженных токенов или паролей
- Кнопка голосования: disable после клика + повторное нажатие заблокировано 
  до ответа сервера (одна логика, не дублировать)
- Голосование невозможно без авторизации на уровне UI
- Нет открытых редиректов

## 6. Доступность (a11y)

- Интерактивные элементы доступны с клавиатуры
- Модальное окно: focus trap, Escape закрывает, aria-modal="true"
- Изображения имеют alt
- Кнопки имеют понятный текст или aria-label
- Контраст #CDDCDC на #0F0F0F и #243434 соответствует WCAG AA

## 7. Производительность

- Изображения кандидатов — lazy loading
- Шрифты подключены с font-display: swap
- Нет блокирующих запросов в рендере
- Компоненты за модалкой: lazy mount (не рендерить пока закрыта)
