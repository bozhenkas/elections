## ПРИОРИТЕТ 1 — нужно в первую очередь

### ARCHIVE-01 · Карточка результатов в архиве (завершённые выборы)
**Статус:** в разработке, нужен финальный дизайн
**Описание:** карточка в архиве для уже прошедших выборов (с результатами). Сейчас реализовано программно по приблизительному макету.
**Требования:**
- Два варианта: простые выборы (председатель) и составные (председатели студсоветов)
- Простые: заголовок + чипы + проценты кандидатов + явка
- Составные (закрытое состояние): заголовок + чипы + «НАЖМИТЕ» + явка (570px)
- Составные (открытое состояние): раскрывается со списком всех студсоветов
- Layout: `flex-direction: column; gap: 40px; padding: 32px 40px`
- Чипы: дата (outline-pink), время (filled pink), год (outline-pink)
- Стиль: `border-radius: 12px; border: 0.5px solid #636363; background: rgba(88, 139, 139, 0.17); backdrop-filter: blur(81.15px)`

### CARD-01 · Карточка предстоящих выборов
**Статус:** реализована, нужна доработка дизайна
**Описание:** главная карточка на лендинге для текущих/предстоящих выборов.
**Требования:**
- Чипы: «предстоящие выборы» (filled), дата (outline), время (filled)
- Нижний ряд: «до голосования» (outline), «12 дней» (filled)
- Все чипы: italic SF Pro Display 21px, border-radius 4px
- Кнопка «перейти к выборам» справа внизу
- Силуэты кандидатов справа

### CANDIDATE-01 · Карточка кандидата (детальная)
**Статус:** есть базовый `CandidateCard.tsx`, нужен полный дизайн
**Описание:** полная страница/модал кандидата с фото, информацией, программой.
**Референс:** текущая реализация на стороннем сайте (фото + личная информация + контакты + быстрые действия + «О кандидате» + предвыборная программа)
**Требования:**
- Фото кандидата (крупное, слева)
- Личная информация: возраст, факультет, курс, группа
- Контакты: VK, Telegram
- Кнопки: «Назад к кандидатам», «К голосованию»
- Блок «О кандидате» (текст)
- Блок «Предвыборная программа» (текст, возможно со свёрткой)
- Glassmorphism стиль, шрифты по дизайн-системе

## Общие требования

- Все компоненты в Figma-файле `T6q4wkkBnkGlMxe8qcCSbq`
- Шрифты:
  - **Serpantin** — только UPPERCASE заголовки
  - **Decor Italic** — только мелкие подписи-акценты
  - **SF Pro Display / Inter** — весь остальной UI (чипы — italic)
- Цвета строго из дизайн-системы (см. `CLAUDE.md`)
- Розовый `#FFCECE` — только бейджи/чипы
- Бордеры `0.5px #636363`
- Скругления: карточки `12px`, чипы `4px`, кнопки pill
- Без теней и градиентов на фоне

## Модель данных

Выборы хранятся через API (`src/api/types.ts`):
- ~2 выборов в год: `chairman` (общий председатель) + `councils` (по студсоветам)
- Кандидат: имя, возраст, факультет, курс, группа, фото, био, программа, контакты
- Результаты: процент, голоса, победитель, явка
- Статусы: `upcoming` → `active` → `completed`
- Админ-панель добавляет выборы через отдельный интерфейс
